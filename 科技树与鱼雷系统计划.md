# 科技树分支 + 鱼雷系统 实现计划

## Context

当前游戏是 3D 海战，10 级线性升级，只有火炮武器。本次需求在 3→4 级时引入三条科技树（驱逐舰/战列舰/巡洋舰），并新增鱼雷武器系统，丰富战术玩法。

---

## 一、舰船职业系统

### 1.1 技术树选择
- 玩家从 3 级升到 4 级时，游戏暂停，弹出职业选择界面（三个卡片：驱逐舰/战列舰/巡洋舰）
- 选择后保存到 DB（`players.ship_class`），后续不可更改（重置进度才重置）
- 已选择过职业的玩家（重进游戏），跳过选择直接用保存的职业

### 1.2 职业配置（`CLASS_CONFIG` in ship.js）

每个职业有独立的 4-10 级配置，覆盖以下属性：

| 属性 | 驱逐舰 | 巡洋舰 | 战列舰 |
|------|--------|--------|--------|
| HP | 低（当前值×0.6） | 中（当前值×1.0） | 高（当前值×1.4） |
| 航速（maxSpeed） | 1.4× 当前值 | 1.0× | 0.7× |
| 转弯半径 turnRadius | 0.7× 当前值 | 1.0× | 1.4× |
| 火炮伤害 | 0.7× | 1.0× | 1.3× |
| 火炮装填 | 1.0× | 0.7×（速射） | 1.2×（慢） |
| 炮塔数量 | 少 | 中 | 多 |
| 鱼雷档位 | 3档（短/中/远） | 仅1档（短） | 无 |
| 鱼雷管数 | 多（4-8管） | 少（2-4管） | 0 |

**1-3 级保持不变**，使用现有 `LEVEL_CONFIG`。

### 1.3 Ship 类改造

- 新增 `shipClass` 属性（null / 'destroyer' / 'cruiser' / 'battleship'）
- `upgradeToLevel(4+)` 根据 `shipClass` 查找 `CLASS_CONFIG[shipClass][level]`
- 新增鱼雷管（`torpedoTubes`）：位置在船体两侧，每管有朝向角（port/starboard ±90°）
- 新增 `maxSpeed` 属性，替代硬编码的 `MAX_SPEED`，由配置决定

## 二、鱼雷武器系统

### 2.1 新文件：`frontend/src/game/torpedo.js`

**TorpedoManager** 类：
- 管理所有在飞行中的鱼雷
- 鱼雷在水面（y≈-0.5）直线运动，无重力抛物线
- 碰撞检测：与敌舰、玩家舰船的 AABB 碰撞
- 到达最大射程自动消失
- 鱼雷视觉效果：细长圆柱体 + 水面尾迹

**鱼雷三档**（基础值，随等级按系数增长）：

| 档位 | 按键 | 速度 | 射程 | 备注 |
|------|------|------|------|------|
| 1（快/短） | 2 | 22.2 m/s (80 km/h) | 400m | 巡洋舰仅有此档 |
| 2（中） | 3 | 16.7 m/s (60 km/h) | 600m | 基准值 |
| 3（慢/远） | 4 | 12.5 m/s (45 km/h) | 800m | 仅驱逐舰 |

**等级缩放**：每升一级，射程×1.05，速度×1.03

### 2.2 发射机制

- 按左键发射当前所有就绪鱼雷管
- 扇形散布：窄扇（±5°）和宽扇（±15°），鱼雷管均匀分布在扇形角度内
- 鱼雷管装填冷却（destroyer ~8s, cruiser ~12s，随等级缩短）
- 鱼雷管朝向固定（在船体两侧），不能 360° 旋转
- 发射角度限制：仅管口朝向 ±45° 范围内可发射

### 2.3 鱼雷辅助瞄准（HUD 覆盖层）

鱼雷模式下在屏幕上显示：
1. **扇形射界**：以船身方向为基准的扇形区域，高亮当前可覆盖范围
2. **白色预瞄线**：锁定最近的屏幕中心敌舰，根据敌舰当前速度/航向计算预测命中点，画白色虚线
3. **武器模式指示器**：左下角显示"火炮"/"鱼雷"，鱼雷模式显示当前档位和扇形模式

## 三、控制系统改造（controls.js）

### 3.1 新增状态

```
weaponMode: 'gun' | 'torpedo'
torpedoTier: 1 | 2 | 3
torpedoSpread: 'narrow' | 'wide'
```

### 3.2 按键映射

| 按键 | 功能 |
|------|------|
| 1 | 切换到火炮模式 |
| 2 | 切换到鱼雷档位1（短/快） |
| 3 | 切换到鱼雷档位2（中），双击切换窄/宽扇 |
| 4 | 切换到鱼雷档位3（长/慢），仅驱逐舰 |

- 双击检测：300ms 内连按两次 3 视为切换扇形模式（而非切换档位）
- 战列舰按 2/3/4 无效果（无鱼雷）
- 巡洋舰按 3/4 无效果（仅有档位1）
- 鼠标左键：火炮模式发射炮弹，鱼雷模式发射鱼雷

## 五、数据库 & API

### 5.1 database.py

- `players` 表新增 `ship_class TEXT DEFAULT NULL`
- 新增 `get_player_ship_class(player_id)` 和 `update_player_ship_class(player_id, ship_class)`

### 5.2 main.py

- 新增 API：
  - `GET /api/players/{id}/class` → 返回 `{"shipClass": "destroyer"|...}`
  - `PUT /api/players/{id}/class` → 保存职业选择
- `GET /api/players/{id}/progress` 返回中增加 `shipClass` 字段

## 六、引擎集成（engine.js）

### 6.1 关键改动

1. `start()` 时从 API 加载 `shipClass`，传给 Ship 构造函数
2. `_checkLevelUp()` 中，如果 `oldLevel === 3 && newLevel === 4 && !shipClass`：
   - 暂停游戏循环（`this.running = false`）
   - 触发 `onClassSelect` 回调，App 层显示选择界面
   - 选择完成后恢复游戏
3. 新增 `TorpedoManager` 实例，在主循环中 `update`
4. `_loop()` 中根据 `controls.weaponMode` 决定发射炮弹还是鱼雷
5. 鱼雷模式下的瞄准信息传递给 HUD

### 6.2 App.jsx 集成

- 新增 `onClassSelect` 回调，显示 `ClassSelectScreen`
- 选择完成后调 API 保存，恢复游戏

## 八、实现顺序

1. **数据库 + API**（database.py, main.py）— 添加 ship_class 字段和接口
2. **ship.js** — 职业配置表 + 鱼雷管 + maxSpeed
3. **torpedo.js**（新建）— 鱼雷系统核心：发射、运动、碰撞、视觉
4. **controls.js** — 武器切换按键（1/2/3/4）、扇形模式
5. **engine.js** — 集成鱼雷管理器、职业选择暂停逻辑
6. **ClassSelectScreen.jsx**（新建）— 职业选择界面
7. **HUD.jsx** — 鱼雷 HUD 覆盖（扇形、预瞄线、装填条）
8. **App.jsx + App.css** — 整合新流程和样式

## 九、验证方式

1. 启动游戏，打到 4 级，确认弹出职业选择界面
2. 分别测试三个职业，验证属性差异（HP、速度、转弯、伤害）
3. 驱逐舰：按 2/3/4 切换鱼雷档位，左键发射，验证扇形/窄扇
4. 巡洋舰：按 2 发射短程鱼雷，按 3/4 无反应
5. 战列舰：按 2/3/4 无反应，只有火炮
6. 鱼雷 HUD：扇形覆盖、预瞄线、装填条
7. 重新登录，确认职业已保存
8. 重置进度后可重新选择职业
