
# 注意事项
- 先写测试再干活，测试通过再交付。测试用例应该包含足够多的情况。
- 用户提出的需求如果你有不清楚的地方， 你需要先和用户讨论，询问清楚具体的需求，问清楚你不懂的点。然后根据任务的复杂度，和用户讨论方案，可以给出几个建议让用户选择，确认方案后再执行。

- 部署事项：完成一次任务后，更新已经部署的docker中的项目（如果在Windows环境中请忽略）
- 如果环境是Windows系统，请使用名为“agent”的conda虚拟环境进行调试，不要新创建环境或者使用base环境，如果在linux系统中请忽略这条。
